# E2E 테스트 계획

## 테스트 계정
- 의사: doctor@myopia-tracker.com / doctor1234!
- 보호자: parent@myopia-tracker.com / parent1234!
- 관리자: admin@myopia-tracker.com / admin1234!

---

## 시나리오 1: 의사 로그인 → 환자 관리 전체 흐름

### 1-1. 로그인
- [ ] 로그인 탭 클릭
- [ ] 이메일/비밀번호 입력 → 로그인
- [ ] 의사 대시보드 표시 확인
- [ ] 네비바에 "홍길동 · OO안과" 표시

### 1-2. 새 환자 등록
- [ ] "새 환자" 클릭
- [ ] 이름: "테스트환자", 생년월일: 2017-06-15, 성별: 남, 관리번호: TEST-001
- [ ] "등록" 클릭
- [ ] 사이드바에 새 환자 표시
- [ ] 자동 생성된 내부 등록번호 (UI에 미표시)

### 1-3. 측정 입력
- [ ] "새 측정" 클릭
- [ ] 모달에 이전 측정값 표시 여부 (첫 측정이면 없음)
- [ ] 날짜: 2025-06-15, OD AL: 22.30, OS AL: 22.25, OD SE: -0.50, OS SE: -0.25
- [ ] "측정 저장" → 동기화 토스트 표시
- [ ] 현재 상태 카드에 값 반영
- [ ] 백분위 자동 계산

### 1-4. 범위 검증 테스트
- [ ] "새 측정" 클릭
- [ ] OD AL에 비정상 값 입력 (예: 2.30)
- [ ] 범위 경고 대화상자 표시 확인
- [ ] 취소 → 값 미저장

### 1-5. 중복 날짜 경고
- [ ] "새 측정" 클릭
- [ ] 같은 날짜 (2025-06-15) 입력
- [ ] 중복 날짜 경고 표시 확인

### 1-6. 두 번째 측정 입력
- [ ] 날짜: 2026-01-15, OD AL: 22.55, OS AL: 22.50, OD SE: -0.75, OS SE: -0.50
- [ ] 저장 후 진행 속도 분석 표시 확인
- [ ] 구간별 진행 속도 테이블 표시
- [ ] 치료 효과 분석 "치료 기록 없음" 표시

### 1-7. 치료 추가
- [ ] "추가" 클릭
- [ ] 치료 종류 선택 (아트로핀 0.01%)
- [ ] 날짜 기본값 오늘 확인
- [ ] 저장 → 치료 태그 "~ 진행중" 표시
- [ ] 성장 차트에 치료 시점 수직선 표시

### 1-8. 치료 종료
- [ ] ⏹ 종료 버튼 클릭
- [ ] 확인 → 태그가 "~ 2026.03.29" 표시
- [ ] 치료 효과 분석 업데이트

### 1-9. 직접 입력 치료
- [ ] "추가" → "직접 입력..." 선택
- [ ] 커스텀 치료명 입력 (예: "마이오가드")
- [ ] 저장 → 태그 표시 확인

### 1-10. 메모 추가/삭제
- [ ] 메모 입력 "경과 관찰 필요"
- [ ] "메모 추가" → 날짜/시간과 함께 표시
- [ ] 삭제 → 메모 제거

### 1-11. 측정 기간 필터
- [ ] "최근 6개월" 클릭 → 필터링 확인
- [ ] "전체" 클릭 → 전체 표시

### 1-12. 환자 정보 수정
- [ ] 수정 아이콘 클릭
- [ ] 이름 변경 → 저장
- [ ] 환자 정보 바 업데이트 확인

### 1-13. CSV 내보내기
- [ ] "CSV" 버튼 클릭 → 파일 다운로드

### 1-14. 리포트 출력
- [ ] "리포트" 버튼 클릭 → 새 창 인쇄 페이지

### 1-15. 차트 이미지 저장
- [ ] "이미지 저장" 클릭 → PNG 다운로드

### 1-16. 전체 내보내기
- [ ] 사이드바 "전체 내보내기" 클릭 → 전체 CSV 다운로드

### 1-17. 환자 삭제
- [ ] "환자 삭제" 클릭
- [ ] 확인 → 환자 제거, 사이드바에서 사라짐

### 1-18. 로그아웃
- [ ] 로그아웃 → 로그인 화면

---

## 시나리오 2: 보호자 로그인 → 자녀 데이터 확인

### 2-1. 로그인
- [ ] 보호자 계정 로그인
- [ ] 보호자 대시보드 표시
- [ ] 자녀 목록 표시 (김민준, 이서연, 최하윤)

### 2-2. 자녀 선택 및 데이터 확인
- [ ] 김민준 클릭
- [ ] 현재 상태 카드 표시 (읽기 전용)
- [ ] 치료 현황 표시 (삭제 버튼 없음)
- [ ] 성장 차트 표시
- [ ] 진행 속도 분석 표시
- [ ] 구간별 진행 속도 표시
- [ ] 측정 기록 표시 (삭제 버튼 없음)
- [ ] "측정 데이터는 담당 안과에서 입력합니다" 안내 표시

### 2-3. 최근 측정일 표시
- [ ] 자녀 카드에 "최근 측정: YYYY.MM.DD" 표시

### 2-4. 자녀 관리
- [ ] "자녀 관리" 클릭
- [ ] 기존 자녀 목록 표시 (수정/삭제 버튼)
- [ ] 새 자녀 추가 (안과 선택 포함)
- [ ] 모달 닫기 → 자녀 목록 업데이트

### 2-5. 리포트 출력
- [ ] "리포트 출력" 클릭 → 새 창 인쇄 페이지

### 2-6. 이미지 저장
- [ ] "이미지 저장" 클릭 → PNG 다운로드

### 2-7. 로그아웃
- [ ] 로그아웃 → 로그인 화면

---

## 시나리오 3: 관리자 로그인 → 시스템 관리

### 3-1. 로그인 및 통계 확인
- [ ] 관리자 계정 로그인
- [ ] 통계 카드 4개 표시 (환자/의사/안과/대기)
- [ ] 수치 정확성 확인

### 3-2. 승인 관리 탭
- [ ] 대기 중인 요청 확인 (있으면 승인/거부 테스트)

### 3-3. 안과 관리 탭
- [ ] 안과 목록 + 의사 수 + 환자 수 표시
- [ ] "새 안과 등록" → 이름 입력 → 등록
- [ ] 수정 → 이름 변경 → 저장
- [ ] 비활성화 → 확인 → 목록에서 제거

### 3-4. 의사 목록 탭
- [ ] 전체 의사 조회
- [ ] 승인 상태 배지 표시
- [ ] "승인 취소" / "비활성화" 버튼 표시

### 3-5. 환자 목록 탭
- [ ] 전체 환자 조회
- [ ] 검색 필터 동작
- [ ] 관리번호 컬럼 표시

### 3-6. 치료 관리 탭
- [ ] 치료 종류 목록 표시
- [ ] "치료 종류 추가" → 이름+색상 → 추가
- [ ] 삭제 → 목록에서 제거

### 3-7. 로그아웃

---

## 시나리오 4: 비로그인 환자 조회

### 4-1. 안과 선택 필수 확인
- [ ] 안과 미선택 → "조회하기" → 에러 메시지

### 4-2. 이름+생년월일 조회
- [ ] 안과: OO안과, 이름: 김민준, 생년월일: 2016-03-15
- [ ] "조회하기" → 환자 결과 화면
- [ ] 현재 상태/치료/차트/측정 기록 표시

### 4-3. 이름+관리번호 조회
- [ ] 안과: OO안과, 이름+관리번호로 검색
- [ ] 검색 성공 확인

### 4-4. 존재하지 않는 환자
- [ ] 이름: 없는환자 → "일치하는 환자를 찾을 수 없습니다"

### 4-5. 레이트 리밋
- [ ] 빠르게 연속 조회 → "잠시 후 다시 시도해주세요"

### 4-6. 돌아가기
- [ ] "← 뒤로" → 로그인 화면

---

## 시나리오 5: 회원가입 플로우

### 5-1. 이메일/비밀번호 검증
- [ ] 비밀번호 8자 미만 → 에러
- [ ] 비밀번호 불일치 → 에러
- [ ] 유효한 입력 → 등록 위저드로 이동

### 5-2. 역할 선택 (Step 1)
- [ ] 보호자/안과의사 선택
- [ ] "다음" → Step 2

### 5-3. 안과 연결 (Step 2)
- [ ] 안과 검색/선택
- [ ] 또는 "새 안과 등록"
- [ ] "다음" → Step 3

### 5-4. 상세 정보 (Step 3)
- [ ] 보호자: 자녀 추가
- [ ] 의사: 이름 입력
- [ ] "가입 완료"

### 5-5. 보호자 가입 완료
- [ ] 로그인 화면으로 이동

### 5-6. 의사 가입 완료
- [ ] 승인 대기 화면 표시
- [ ] 로그아웃만 가능

---

## 시나리오 6: 데이터 흐름 검증 (역할 간)

### 6-1. 의사가 입력한 데이터 → 보호자 확인
- [ ] 의사: 환자에 새 측정 입력
- [ ] 보호자: 같은 환자의 측정 데이터 확인
- [ ] 수치 일치 확인

### 6-2. 의사가 치료 추가 → 보호자 확인
- [ ] 의사: 치료 추가
- [ ] 보호자: 치료 현황에 표시 확인

### 6-3. 의사가 환자 추가 → 관리자 확인
- [ ] 의사: 새 환자 등록
- [ ] 관리자: 통계 환자 수 증가 + 환자 목록에 표시

### 6-4. 관리자 안과 이름 변경 → 의사 네비바 반영
- [ ] 관리자: 안과 이름 변경
- [ ] 의사: 재로그인 후 네비바에 새 이름 표시

---

## 시나리오 7: 에러 핸들링

### 7-1. 잘못된 로그인
- [ ] 틀린 비밀번호 → 한국어 에러 메시지

### 7-2. 빈 폼 제출
- [ ] 각 폼에서 빈 상태로 제출 → 적절한 에러 메시지

### 7-3. 중복 환자 등록
- [ ] 같은 이름+생일 환자 등록 시도 → 에러

### 7-4. 중복 관리번호
- [ ] 같은 관리번호 입력 → 에러

---

## 시나리오 8: 보안 검증

### 8-1. 인증 가드
- [ ] 비로그인 상태에서 #doctor 직접 접근 → 로그인으로 리다이렉트

### 8-2. 미승인 의사
- [ ] 미승인 의사 로그인 → 승인 대기 화면으로 이동

### 8-3. Enter 키 지원
- [ ] 로그인 비밀번호에서 Enter → 로그인
- [ ] 환자조회에서 Enter → 검색
- [ ] 회원가입 비밀번호확인에서 Enter → 가입
