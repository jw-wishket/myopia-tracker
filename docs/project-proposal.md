# 근시관리 트래커 - 프로젝트 제안서

---

## 1. 프로젝트 개요

안과에서 소아·청소년 환자의 근시 진행을 체계적으로 추적하고 관리할 수 있는 **웹 기반 서비스**입니다.

제공해주신 레퍼런스 사이트를 분석하여 **모든 기능을 그대로 구현**하였으며, 실제 안과 업무 환경에 맞춰 **추가 기능과 편의성을 보강**하였습니다.

현재 아래 주소에서 바로 확인하실 수 있습니다:

> **https://myopia-tracker.vercel.app**

| 역할 | 테스트 계정 | 비밀번호 |
|------|-----------|----------|
| 의사 | doctor@myopia-tracker.com | doctor1234! |
| 보호자 | parent@myopia-tracker.com | parent1234! |
| 관리자 | admin@myopia-tracker.com | admin1234! |

샘플 환자 데이터가 입력되어 있어 **로그인 즉시 모든 기능을 체험**하실 수 있습니다.

## 2. 운영 비용

### 현재: 무료

이 서비스는 **Vercel**(웹사이트 호스팅)과 **Supabase**(데이터베이스)의 무료 요금제로 운영됩니다.

| 서비스 | 무료 제공량 | 현재 사용량 |
|--------|-----------|-----------|
| **Vercel** (웹사이트) | 무제한 배포, 100GB 트래픽/월 | 극소량 |
| **Supabase** (데이터베이스) | 500MB 저장, 50,000 사용자/월 | 극소량 |

### 사용자가 늘어나면?

안과 수십 곳, 환자 수천 명 규모까지는 **무료 요금제로 충분**합니다.

그 이상으로 성장할 경우에도 비용은 매우 낮습니다:

| 규모 | Vercel | Supabase | 월 합계 |
|------|--------|----------|--------|
| 안과 1~10곳 (현재) | 무료 | 무료 | **0원** |
| 안과 10~50곳 | 무료 | $25/월 (~3.5만원) | **~3.5만원** |
| 안과 50~200곳 | $20/월 (~2.8만원) | $25/월 (~3.5만원) | **~6.3만원** |

> 별도의 서버 구매, 서버실 관리, IT 인력 없이 운영됩니다.
> 사용자가 적을 때는 비용이 0원이고, 사용자가 늘어난 만큼만 비용이 발생하는 구조입니다.

## 3. 사이트 구성 방식

### 누구나 접속 가능한 웹사이트

별도 프로그램 설치 없이, **인터넷 브라우저(Chrome, Safari 등)에서 주소를 입력하면 바로 사용**할 수 있습니다. 스마트폰, 태블릿, PC 모두 지원됩니다.

### 세 종류의 사용자

```
┌─────────────────────────────────────────┐
│              근시관리 트래커              │
│         (인터넷 웹사이트)                │
├──────────┬──────────┬───────────────────┤
│  의사     │  보호자   │  관리자           │
│          │          │                   │
│ 환자 등록 │ 자녀 조회 │ 의사 승인         │
│ 측정 입력 │ 차트 확인 │ 안과 관리         │
│ 치료 기록 │ 리포트   │ 전체 통계         │
│ 분석 확인 │ (읽기만) │ 치료 종류 관리    │
└──────────┴──────────┴───────────────────┘
         │                    │
         ▼                    ▼
┌─────────────────┐  ┌─────────────────┐
│  Supabase       │  │  Vercel         │
│  (데이터 저장)   │  │  (웹사이트 운영) │
│  클라우드 DB     │  │  자동 배포      │
│  보안 관리       │  │  전세계 접속    │
└─────────────────┘  └─────────────────┘
```

### 데이터 보관 방식

- 모든 데이터는 **Supabase 클라우드 데이터베이스**에 저장됩니다
- 데이터베이스는 **서울 리전(Northeast Asia)**에 위치하여 빠른 응답 속도를 제공합니다
- 데이터는 **암호화**되어 저장되며, **역할별 접근 제한**(의사는 자기 안과만, 보호자는 자기 자녀만)이 데이터베이스 수준에서 적용됩니다
- 모든 데이터 변경(등록, 수정, 삭제)은 **자동으로 기록**(감사 로그)됩니다

## 4. 레퍼런스 기능 구현 현황

레퍼런스 사이트의 **모든 기능을 100% 구현**하였습니다:

| 레퍼런스 기능 | 구현 |
|-------------|:----:|
| 환자 등록 (이름, 생년월일, 성별) | O |
| 안축장(AL) / 굴절력(SE) 양안 측정 입력 | O |
| 백분위 자동 계산 (4~18세, 남/여 참조 데이터) | O |
| 성장 곡선 차트 (백분위 참조선 7개) | O |
| 진행 차트 (안축장/굴절력 시간별 추이) | O |
| 치료 기록 (아트로핀, 드림렌즈, 마이사이트 등) | O |
| 치료 시점 차트 표시 (수직선 annotation) | O |
| CSV 데이터 내보내기 | O |
| 의사 전용 환자 관리 대시보드 | O |
| 보호자 읽기 전용 조회 | O |
| 관리자 의사 승인/거부 | O |
| 환자 검색 (이름+생년월일) | O |
| 회원가입 (역할 선택 → 안과 연결 → 정보 입력) | O |
| 반응형 디자인 (데스크탑/모바일) | O |

## 5. 추가된 기능

레퍼런스에 없지만 **실제 운영에 필요하여 추가한 기능**들입니다:

### 의사를 위한 기능

| 기능 | 설명 |
|------|------|
| **진행 속도 자동 분석** | 6개월/1년/전체 기간별 안축장 변화율을 자동 계산하고 안정/느림/보통/빠름 4단계로 표시 |
| **치료 효과 자동 비교** | 치료 시작 전과 후의 진행 속도를 비교하여 "효과 있음/없음"을 자동 판정 |
| **빠른 진행 경고** | 진행이 빠른 환자를 빨간 경고로 즉시 표시 → 치료 변경이 필요한 환자를 놓치지 않음 |
| **추적 검사 알림** | 측정 입력 시 다음 검사일을 자동 설정, 기한이 지난 환자를 목록으로 표시 |
| **측정값 검증** | 비정상적인 값 입력 시 경고 (소수점 실수 등 방지) |
| **환자 메모** | 환자별로 의사 메모를 기록하고 관리 |
| **키보드 단축키** | N(새 측정), P(새 환자), S(검색) 등으로 빠른 입력 |
| **데이터 가져오기** | 기존 데이터를 CSV 파일로 일괄 등록 |
| **전체 데이터 내보내기** | 안과의 모든 환자 데이터를 한번에 CSV로 내보내기 |
| **인쇄용 리포트** | 환자 기록을 인쇄하거나 PDF로 저장 (전원/보험 제출용) |
| **차트 이미지 저장** | 성장 차트를 PNG 이미지로 다운로드 |

### 보호자를 위한 기능

| 기능 | 설명 |
|------|------|
| **자녀 관리** | 여러 자녀를 등록하고, 다른 안과에 다니는 자녀도 한 화면에서 관리 |
| **새 데이터 알림** | 의사가 새 측정을 입력하면 "NEW" 표시로 알림 |
| **진행 분석 조회** | 의사와 동일한 분석 결과를 읽기 전용으로 확인 |
| **리포트 출력** | 다른 병원 방문 시 기록을 인쇄하여 공유 가능 |

### 관리자를 위한 기능

| 기능 | 설명 |
|------|------|
| **안과 관리** | 안과 등록/수정/비활성화, 소속 의사 수·환자 수 확인 |
| **의사 관리** | 전체 의사 조회, 승인 취소, 계정 비활성화 |
| **환자 목록** | 전체 환자 조회 및 검색 |
| **치료 종류 관리** | 새로운 치료법 추가/삭제, 색상 지정 |
| **통계 대시보드** | 전체 환자/의사/안과/대기 현황 한눈에 확인 |

### 편의성 개선

| 기능 | 설명 |
|------|------|
| **치료 시작~종료 관리** | 치료를 언제 시작하고 언제 종료했는지 기록 |
| **관리번호** | 안과별 자체 관리번호 부여 (자동 등록번호와 별도) |
| **비밀번호 변경/찾기** | 앱 내에서 바로 처리 |
| **중복 환자 방지** | 같은 이름·생년월일 환자 중복 등록 차단 |
| **날짜 입력 제한** | 잘못된 연도(6자리 등) 입력 방지 |
| **Enter 키 지원** | 입력 후 Enter로 바로 실행 |
| **로딩 표시** | 데이터 저장/삭제 시 진행 상태 표시 |

## 6. 보안

의료 데이터를 다루는 서비스인 만큼, **20가지 보안 취약점을 검토하고 모두 수정**하였습니다:

| 보안 조치 | 설명 |
|----------|------|
| 로그인 인증 | 이메일+비밀번호 인증 필수 |
| 역할별 접근 제한 | 의사는 자기 안과만, 보호자는 자기 자녀만 |
| 데이터베이스 수준 보안 | 서버에서 접근 권한을 강제 (클라이언트 조작 불가) |
| 의사 승인 제도 | 관리자 승인 전까지 의사 기능 사용 불가 |
| 감사 로그 | 모든 데이터 변경 내역 자동 기록 |
| 악성 코드 방지 | 입력값 검증 + 화면 출력 시 안전 처리 |
| 보안 헤더 | 브라우저 수준의 추가 보호 장치 적용 |
| 계정 비활성화 | 퇴사 의사의 접근을 즉시 차단 가능 |

## 7. 진행 방식 제안

### 현재 상태

사이트는 이미 **개발 완료 및 배포**되어 있습니다. 테스트 계정으로 모든 기능을 직접 확인하실 수 있습니다.

### 제안 프로세스

```
현재 상태                    다음 단계
───────────────────         ──────────────────────

[1] 사이트 확인              테스트 계정으로 접속하여
    및 기능 테스트            모든 기능을 직접 체험

         ↓

[2] 피드백 및                추가 기능 요청,
    추가 요구사항             수정 사항 전달

         ↓

[3] 보완 작업                피드백 반영 후
                            즉시 사이트에 반영

         ↓

[4] 운영 시작                커스텀 도메인 연결,
                            실제 환자 데이터 입력 시작
```

### 운영 시작까지 필요한 작업

| 단계 | 작업 | 예상 시간 |
|------|------|----------|
| 1 | 기능 검토 및 피드백 | 고객 측 |
| 2 | 피드백 반영 수정 | 내용에 따라 상이 |
| 3 | 커스텀 도메인 연결 (선택) | 30분 |
| 4 | 테스트 데이터 정리 + 실제 계정 생성 | 1시간 |
| 5 | 운영 시작 | 즉시 |

> 별도의 서버 구축이나 설치 과정이 없으므로, **피드백 반영 후 바로 운영**이 가능합니다.
